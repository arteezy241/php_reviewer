PHP Functions & Modular Programming

1. What is a function?

A function is a named, reusable block of code that performs a specific task. It can accept input (parameters) and optionally return a result (return value).

Why use functions?

  • Reusability — write once, call many times
  • Organization — breaks code into smaller, logical pieces
  • Easier debugging — errors are isolated to specific functions
  • Simpler maintenance — update the function, and the change applies everywhere it's used
  • Efficiency — PHP compiles each function only once per page load
Mnemonic — RODEM: Reusability · Organization · Debugging · Easy maintenance · More efficient

Types of functions

  • Built-in Pre-made by PHP — e.g., strlen(), array_push(), date()
  • User-defined Functions you write yourself for your project's specific needs
Section summary: Functions = named blocks of reusable code. PHP has built-in ones, and you can define your own.

2. Creating and calling functions

Naming rules

  • Use the function keyword to declare it
  • Names are NOT case-sensitive — myFunc() = MYFUNC()
  • Must start with a letter or underscore, never a digit
  • Followed by parentheses () and curly braces {}

Common mistake: Starting a function name with a number like 2doSomething() — this will cause a syntax error.

// Declaration
function sayHello() {
    echo "Hello, IT Students!";
}

// Calling it
sayHello(); // Output: Hello, IT Students!

Exam tip: Declaring a function does NOT run it. You must call it separately.

Section summary: Declare with function name() {}. Call it by writing name(). Case doesn't matter.

3. Parameters and arguments

Parameter = the variable placeholder in the function definition.
Argument = the actual value you pass in when calling the function.

Memory trick: "Parameters are in the Plan (definition), Arguments are the Actual values."

  • Multiple parameters are separated by commas
  • Default values — if no argument is given, PHP uses the default
function greetUser($name = "Guest") {
    echo "Welcome, $name!";
}

greetUser("Alyssa"); // Output: Welcome, Alyssa!
greetUser();         // Output: Welcome, Guest! (uses default)

Common mistake: Putting a parameter with a default value before one without a default. Always put defaults at the end: function test($a, $b = "default")

Section summary: Parameters = placeholders. Arguments = real values. Add default values so the function still works if an argument is omitted.

4. Returning values

Use return when you want the function to give back a computed value instead of just printing it.

  • return immediately stops the function — no code after it runs
  • Can return any data type: string, integer, float, array, object, boolean
  • You can store the returned value in a variable
function addNumbers($num1, $num2) {
    $result = $num1 + $num2;
    return $result;
}

$sum = addNumbers(5, 10);
echo $sum; // Output: 15

Exam tip: A function with no return statement returns NULL by default.

Section summary: Use return to get a value out of a function and stop its execution.

5. Pass by value vs. pass by reference

FeaturePass by Value (default)Pass by Reference (&)
What's passedA copy of the variableThe actual memory address
Changes inside functionDo NOT affect the originalDO affect the original
Syntaxfunction add($n)function add(&$n)
// Pass by Value — original unchanged
function addFiveVal($num) {
    $num += 5;
}
$n = 10;
addFiveVal($n);
echo $n; // Still 10

// Pass by Reference — original IS changed
function addFiveRef(&$num) {
    $num += 5;
}
$n = 10;
addFiveRef($n);
echo $n; // Now 15

Exam tip: The & symbol goes in the function definition (before the parameter), not in the function call.

Section summary: By value = safe copy. By reference (&) = edits the original variable.

6. Variable scope

Scope = where a variable can be accessed.

  • Local scope — variable created inside a function; only accessible there
  • Global scope — variable created outside all functions; NOT accessible inside a function by default
  • global keyword — used inside a function to explicitly access a global variable
$x = 10; // Global variable

function testScope() {
    global $x;  // Now we can use $x
    echo $x;    // Output: 10
}

testScope();

Common mistake: Trying to use a global variable inside a function without the global keyword — PHP will treat it as a separate, undefined local variable.

What about superglobals? Variables like $_GET, $_POST, $_FILES, and $_SESSION are always accessible everywhere — no global keyword needed!

Section summary: Variables live in bubbles. Use global to reach outside a function's bubble. Superglobals have no bubble — they're always available.

HTML Forms & PHP Input Validation

1. HTML form basics

An HTML <form> is the standard way to collect user input and send it to a server for processing.

  • <form> — the wrapper for all inputs; defines how data is sent
  • <label> — links a text label to an input using for attribute; improves accessibility and usability (clicking the label focuses the input)
  • <input>, <select>, <textarea> — different input field types
<label for="email">Email:</label>
<input type="email" id="email" name="user_email">

Exam tip: The for attribute on <label> must match the id attribute on the input — not the name.

4. PHP validation and sanitization

Golden rule: Never trust user input! Malicious users can inject scripts (XSS attacks) through form fields.

isset() vs. empty()

  • isset($var) — returns true if the variable exists AND is not NULL
  • empty($var) — returns true if the variable is "", 0, NULL, false, or an empty array

Exam tip: isset("0") returns true (it exists), but empty("0") returns true (0 is considered empty). This is a common trap!

Sanitization functions

  • trim($str) — removes whitespace, tabs, newlines from both ends
  • stripslashes($str) — removes backslashes used to escape characters
  • htmlspecialchars($str) — converts <, >, & into safe HTML entities — key XSS prevention tool
function sanitizeInput($data) {
    $data = trim($data);
    $data = stripslashes($data);
    $data = htmlspecialchars($data);
    return $data;
}

$name = sanitizeInput($_POST['name']);
Remember the order — TSH: Trim → Stripslashes → Htmlspecialchars
Section summary: Always validate (check the data) and sanitize (clean the data) before using any user input. Use htmlspecialchars() to block XSS attacks.

PHP File Handling & Secure File Uploads

1. Why handle files?

Unlike variables that disappear when a script ends, files persist on the server. PHP can read, write, create, and delete files — useful for logs, configs, and user-uploaded data.

2. Opening and closing files

Always open a file before using it and close it when done to free server resources.

  • fopen($filename, $mode) — opens a file and returns a file handle (pointer)
  • fclose($handle) — always close after you're done

Common modes — know these!

ModeNameWhat it does
'r'ReadOpens for reading. File must exist. Starts at beginning.
'w'WriteOpens for writing. Erases existing content! Creates file if not found.
'a'AppendAdds to the end. Does NOT erase content. Creates file if not found.
'r+'Read+WriteOpens for both. File must exist.
'w+'Write+ReadRead and write. Erases existing content.

Watch out: Mode 'w' deletes all existing content in the file instantly! Use 'a' if you want to add to a file without destroying it.

Mode trick — "R eads, W ipes, A ppends"

3. Reading and writing data

Reading functions

  • fread($handle, $size) — reads a specific number of bytes
  • fgets($handle) — reads one line at a time (great for large files)
  • file_get_contents($filename) — reads the entire file into a single string (easiest method, no handle needed)

Writing function

  • fwrite($handle, $string) — writes text to a file based on the mode
// Quick read — entire file at once
$content = file_get_contents("data.txt");
echo $content;

// Line-by-line read — better for large files
$handle = fopen("data.txt", "r");
while (!feof($handle)) {
    $line = fgets($handle);
    echo $line;
}
fclose($handle);

// Writing
$handle = fopen("log.txt", "a");
fwrite($handle, "New log entry\n");
fclose($handle);

Exam tip: feof() means "end of file" — use it in a while loop to stop reading when you reach the end.

Section summary: Use file_get_contents() for simple reads. Use fgets() for line-by-line. Use fwrite() with mode 'a' to append safely.

4. Secure file uploads

Required setup

  1. Form method must be "POST"
  2. Form enctype must be "multipart/form-data"
  3. Include an <input type="file"> field
<form method="POST" enctype="multipart/form-data">
    <input type="file" name="userfile">
    <input type="submit" value="Upload">
</form>

The $_FILES superglobal

When a file is uploaded, PHP stores its info in $_FILES:

  • $_FILES['userfile']['name'] — original filename from user's computer
  • $_FILES['userfile']['tmp_name'] — temporary server path where PHP stored it
  • $_FILES['userfile']['size'] — file size in bytes
  • $_FILES['userfile']['type'] — MIME type (e.g., image/jpeg)
  • $_FILES['userfile']['error'] — error code (0 = no error)

Exam tip: The file is first stored in a temporary folder. You must use move_uploaded_file() to move it to its permanent location.

$tmpPath = $_FILES['userfile']['tmp_name'];
$destination = "uploads/" . $_FILES['userfile']['name'];
move_uploaded_file($tmpPath, $destination);

5. Security checks for uploads

Never let users upload any file they want without validation!

  • Check if file already exists: file_exists($path)
  • Check file size: $_FILES['userfile']['size'] <= 500000 (500KB limit)
  • Check file type/extension: Use an allowlist (only .jpg, .png, .pdf, etc.)
$allowedTypes = ['jpg', 'jpeg', 'png', 'gif'];
$ext = strtolower(pathinfo($_FILES['userfile']['name'], PATHINFO_EXTENSION));

if (!in_array($ext, $allowedTypes)) {
    echo "Error: Invalid file type.";
} elseif ($_FILES['userfile']['size'] > 500000) {
    echo "Error: File too large.";
} elseif (file_exists($destination)) {
    echo "Error: File already exists.";
} else {
    move_uploaded_file($_FILES['userfile']['tmp_name'], $destination);
    echo "Upload successful!";
}
Upload security checklist — "TEN": Type check · Existence check · Number/size check
Section summary: File uploads need POST + multipart enctype. Use $_FILES to get file info. Always validate type, size, and name before calling move_uploaded_file().
